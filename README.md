# LLM Benchmark Studio

ローカル LLM のエージェント能力を体系的に評価するためのベンチマークツールです。
**FastAPI バックエンド + バニラ JS SPA** で構成された Web UI から、LM Studio 上のモデルに対してベンチマークを実行・採点・比較できます。

## 機能一覧

| 機能 | 説明 |
|------|------|
| **ベンチマーク実行** | LM Studio 上のモデルに対してプロンプトを順次送信し、回答を収集 |
| **ライブ出力表示** | 実行中のトークンストリームをリアルタイム表示、各問の経過時間も計測 |
| **採点 UI** | 0 / 1 / 3 / 5 の4段階で手動採点、採点コメントも記録可能 |
| **HTML レポート** | 採点済み結果から見やすい HTML レポートを自動生成・保存 |
| **Benchmark タブ** | ベンチマーク定義（問題文・採点基準）をブラウザ上で閲覧 |
| **結果フィルタリング** | モデル名・ベンチマーク・スコア閾値で結果一覧を絞り込み |
| **プリセット** | よく使うモデル設定（temperature, top_p など）を保存・呼び出し |
| **問題の再実行** | 特定の問題だけを選択して再実行し、既存結果に上書き保存 |
| **ウォームアップ** | 最初の問題の実行前にモデルロード用のダミーリクエストを自動送信し、計測時間を正確化 |

## 前提条件

- **Python 3.11 以上**
- **[LM Studio](https://lmstudio.ai/) 0.3 以上** がインストール済みであること
- LM Studio に評価したいモデルがダウンロード済みであること

## LM Studio との接続

### 1. LM Studio でローカルサーバーを起動する

1. LM Studio を起動し、左サイドバーの **「Developer」** タブを開く
2. **「Start Server」** ボタンを押してローカルサーバーを起動する
3. デフォルトでは `http://localhost:1234` でリッスン開始

> **ポートを変更した場合:** `app.py` の先頭にある `LM_STUDIO_BASE` 変数を書き換えてください。
> ```python
> LM_STUDIO_BASE = "http://localhost:1234"  # ← ポートを変更
> ```

### 2. モデルをロードする

1. LM Studio の **「My Models」** からモデルを選択
2. **「Load Model」** をクリックしてモデルをメモリにロード
3. 右上のステータスが **「Running」** になれば準備完了

### 3. 接続確認

Web UI を起動すると、左パネル上部の **「LM Studio」** 欄に接続状態とロード済みモデルが表示されます。
`⚡ Connected` と表示されていれば正常に接続されています。

## ベンチマーク実行

### 基本的な実行手順

1. **左パネル**で以下を設定する
   - **Model:** ドロップダウンから LM Studio にロードされているモデルを選択
   - **Benchmark:** 実行するベンチマーク定義ファイルを選択
   - **System Prompt:** 必要に応じて変更（デフォルトで動作）
   - **Temperature / Max Tokens:** 生成パラメータを設定

2. **「▶ Run」ボタン** を押して実行開始

3. **ライブ出力エリア** に各問の進捗がリアルタイム表示される
   - 実行前に `モデルをロード中...` が表示される（ウォームアップ）
   - 各問の経過時間が問題完了時に表示される

4. 全問完了すると `Complete` バナーと出力ファイル名が表示される

### 生成パラメータ

| パラメータ | 説明 | 推奨値 |
|-----------|------|--------|
| Temperature | 出力のランダム性（高いほど多様） | `0.7` |
| Max Tokens | 1回答の最大トークン数 | `2048`〜`4096` |
| Top P | nucleus sampling の確率閾値 | 未指定 |
| Top K | 上位 K トークンからサンプリング | 未指定 |
| Frequency Penalty | 既出トークンへのペナルティ | 未指定 |
| Repeat Penalty | 繰り返しへのペナルティ | 未指定 |
| Min P | 最低確率閾値 | 未指定 |
| Seed | 乱数シード（再現性確保） | 未指定 |
| Stop Tokens | 生成を打ち切るトークン列 | 未指定 |
| Interval (sec) | 問題間の待機時間（API 負荷対策） | `3.0` |

### 問題を選択して再実行

結果詳細の各問題カード右上にある **🔄（再実行）ボタン** を押すと、その問題だけを再実行して既存結果に上書きできます。採点コメントとスコアは引き継がれます。

## 結果の確認と採点

### Results タブ

**中央パネル** に実行済み結果がカードで一覧表示されます。

- **スコア（数値 / 5）**: 採点済み問題の平均スコア
- **進捗バー**: 採点済み問題数 / 全問数
- カードをクリックすると **右パネル** に詳細が展開

### 詳細パネル

右パネルには以下の情報が表示されます。

| セクション | 内容 |
|-----------|------|
| **統計サマリー** | 平均スコア・採点済み問数・モデル名 |
| **実行パラメータ** | temperature, max_tokens など実行時の設定値 |
| **ドメイン別問題一覧** | 各問のレスポンス・採点基準・採点ボタン |

### 採点方法

各問題カードの **採点ボタン（0 / 1 / 3 / 5）** をクリックするとスコアが保存されます。

| スコア | 基準 |
|--------|------|
| **5** | 全要件を完全に満たす |
| **3** | おおむね正しいが部分的に不十分 |
| **1** | 方向性は合っているが重要な要件を欠く |
| **0** | 要件を無視または完全に誤っている |

採点ボタン下の **テキストエリア** に採点コメント（判定根拠・失敗パターン）を記入できます。フォーカスを外すと自動保存されます。

全問採点が完了すると、ファイルが自動的に `results/unscored/` から `results/scored/` に移動します。

### HTML レポート

採点済みの結果は **「Report」ボタン** から HTML レポートを別タブで開けます。レポートは `results/scored/` に `.html` ファイルとして自動保存されます。

## ベンチマーク内容の確認

上部の **「Benchmark」タブ** で、ベンチマーク定義ファイルの内容をブラウザ上で閲覧できます。

- **左サイドバー**: ベンチマーク一覧からファイルを選択
- **メインエリア**: ドメイン・問題のツリー表示
  - 問題をクリックすると **プロンプト全文** と **4段階の採点基準** が展開表示
  - 採点基準はスコアに対応した色でハイライト

ベンチマーク定義ファイルは `benchmarks/` ディレクトリに `.json` 形式で保存します。

---

## 結果のフィルタリング

Results タブの中央パネル上部の **フィルターバー** で結果を絞り込めます。

| フィルター | 動作 |
|-----------|------|
| **Model（テキスト）** | モデル名に入力文字列が含まれる結果のみ表示 |
| **Benchmark（選択）** | 指定したベンチマークの結果のみ表示 |
| **Score（選択）** | 平均スコアが指定閾値以上の結果のみ表示 |

## Claude スキル

このプロジェクトは **Claude Code** 上で以下のスキルが使えます。

### `/score` — 一括採点

```
採点して
```

`results/unscored/` にある全ての未採点ファイルを読み込み、各問の `response` を `scoring_criteria` と照合してスコア（0/1/3/5）と採点コメントを付与し、全問採点完了後に `results/scored/` へ移動します。

### `/build-benchmark [ファイル名]` — ベンチマーク生成

```
ベンチマークを作成して
```

`benchmarks/draft/` に置いたドラフトファイル（形式自由）を読み込み、JSON スキーマ準拠のベンチマーク定義ファイルを生成して `benchmarks/` に保存します。

### `/compare [ベンチマークファイル名]` — モデル比較レポート

```
form_detect_benchmark.json のモデルを比較して
```

指定ベンチマークの全採点済み結果を横断し、以下を含む Markdown 比較レポートを生成して `results/reports/` に保存します。

- 全体・ドメイン別スコア比較表
- 各モデルの強み・弱み・速度特性・出力制御精度
- ユースケース別推奨（精度優先 / 速度優先 / バランス）
- 総合推奨モデルと選定理由

## よくある問題

**Q: 「LM Studio に接続できません」と表示される**
→ LM Studio の Developer タブでサーバーが起動しているか確認してください。ポートがデフォルト（1234）と異なる場合は `app.py` の `LM_STUDIO_BASE` を修正してください。

**Q: モデル一覧に何も表示されない**
→ LM Studio でモデルをロードしてから Web UI を再度確認してください（左パネルの更新ボタンを使用）。

**Q: 最初の問題の経過時間が極端に長い**
→ ウォームアップ機能が有効になっており、モデルロード時間は除外されているはずです。LM Studio のサーバーが起動してから少し待ってから実行してみてください。

**Q: 採点が完了してもファイルが `scored/` に移動しない**
→ 全問に対してスコアが設定されているか確認してください。1問でも `null` が残っていると移動しません。
