# /score — 未採点ベンチマーク結果の採点

`results/unscored/` にある全ての未採点結果ファイルを読み込み、各問題に対してスコアと採点コメントを付与する。

## 実行手順

### 1. 対象ファイルの列挙

`results/unscored/*.json` を全て列挙する。ファイルが存在しない場合は「採点対象なし」と報告して終了。

### 2. 各問題の採点

ファイル内の全ドメイン・全問題を走査し、`"score": null` の問題を採点する。

**採点方法:**
- `response` の内容を `scoring_criteria` の各基準と照合する
- **0 / 1 / 3 / 5** の4段階から最も当てはまるものを選ぶ
- `response` が `null`（エラー）の場合は **0** とする
- 数値条件（「8個以上」等）は厳密に判定する
- 「軽微な誤り」「不完全」の表現は回答の完成度で判断する

**スコアの意味:**
| score | 意味 |
|------:|------|
| 5 | 全要件を完全に満たす |
| 3 | おおむね正しいが部分的に不十分 |
| 1 | 方向性は合っているが重要な要件を欠く |
| 0 | 要件を無視または完全に誤っている |

**採点コメントの方針:**
- スコアが 5 の場合: 何が優れていたかを 1 行で記す（例:「正確なパスのみを返した」）
- スコアが 3 以下の場合: 減点理由を具体的に 1〜2 文で記す（例:「/reach-out を選択したが説明文を付加した。出力制御の指示を無視している」）
- `response` が `null` の場合: 「APIエラーのため回答なし」と記す

### 3. JSON ファイルの更新

Edit ツールまたは Python スクリプトで該当箇所を書き換える:

```
"score": null,         →  "score": <値>,
"score_comment": null, →  "score_comment": "<採点コメント>",
```

既存ファイルに `score_comment` フィールドが存在しない場合は、`score` の直後に追加する。

### 4. 採点完了後の移動

ファイル内の全問題が採点済みになったら:
```bash
mv results/unscored/<filename>.json results/scored/
```

### 5. 結果の報告

以下の形式でまとめて出力する:

```
📋 <ファイル名>  (<モデル名>)
  ├─ <q_id>  <タイトル>  → <score>/5  (<採点コメント>)
  ├─ ...
  └─ Avg: <平均>/5  [→ scored/ へ移動]
```

複数ファイルがある場合はファイルごとに区切って報告する。
