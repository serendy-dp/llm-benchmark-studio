# 実践チュートリアル — モデル評価の一連の流れ

LM Studio のモデル接続から、ベンチマーク作成・実行・採点・比較レポート閲覧まで、**実際のユースケースを題材に一通りの操作を体験**するチュートリアルです。

---

## このチュートリアルで体験すること

```
①LM Studio 接続確認
      ↓
②ドラフトを書く（評価したいことを自由記述）
      ↓
③ /build-benchmark でベンチマーク JSON を自動生成
      ↓
④Web UI でモデル A を実行
      ↓
⑤採点（UI 手動 または /score スキル）
      ↓
⑥モデル B でも同じベンチマークを実行・採点
      ↓
⑦ /compare でモデル比較レポートを生成
      ↓
⑧Reports タブで結果を閲覧
```

> **使用ツール**
> - LM Studio（ローカル LLM サーバー）
> - LLM Benchmark Studio（このアプリ）— `http://localhost:8001`
> - Claude Code（ベンチマーク生成・採点・比較レポート生成）

---

## ① LM Studio の起動と接続確認

### LM Studio の準備

1. LM Studio を起動し、左サイドバーの **「Developer」** タブを開く
2. **「Start Server」** をクリック → `http://localhost:1234` で起動
3. **「My Models」** から評価したいモデルを選んで **「Load Model」**
4. ステータスが **「Running」** になるまで待つ

> **今回の題材で使用するモデル（例）**
> - モデル A: `qwen2.5-7b-instruct`（精度重視）
> - モデル B: `lfm2.5-1.2b`（速度重視）
>
> ※ 手元にあるモデルで問題ありません。2つ用意すると比較が楽しめます。

### アプリの起動

```bash
# リポジトリのディレクトリで
source .venv/bin/activate
uvicorn app:app --port 8001 --reload
```

ブラウザで `http://localhost:8001` を開き、右上のインジケータが **緑（Connected）** になっていれば準備完了です。

## ② ドラフトを書く

評価したい内容を **自由な形式で** `benchmarks/draft/` に書きます。JSON の知識は一切不要です。

### ドラフトを作成する

`benchmarks/draft/customer_support_draft.md` を作成してください（以下はサンプル内容）。

```markdown
# カスタマーサポート AI 応答品質ベンチマーク

## 目的
ECサイトのカスタマーサポートAIとして、ユーザーの問い合わせに適切に応答できるかを評価する。

## 評価したいこと

### 問い合わせ意図の理解
- 「注文した商品がまだ届かない」→ 配送遅延の問い合わせと正しく理解できるか
- 「返品したいんですが」→ 返品手続きへ誘導できるか
- 「ポイントの使い方がわからない」→ ポイント機能の質問と判断できるか

### 適切な次ステップ提示
- 問題解決のために必要な情報（注文番号など）を聞き返せるか
- 担当部署への転送が必要な場面を判断できるか

### トーン・礼節
- 敬語が正しく使われているか
- ユーザーが怒っているときに落ち着いた対応ができるか

## 難易度のイメージ
- 簡単: 単純な FAQ 的問い合わせ
- 普通: 複数の意図が混在する問い合わせ
- 難しい: 怒っているユーザー、曖昧な問い合わせへの対応
```

**ポイント:** 箇条書きでも文章でも構いません。「何を測りたいか」が伝われば、あとは Claude が補完します。

---

## ③ /build-benchmark でベンチマーク JSON を生成

Claude Code のターミナルで以下を実行します。

```
/build-benchmark customer_support_draft.md
```

または自然言語でトリガーすることもできます。

```
benchmarks/draft/customer_support_draft.md を読んでベンチマークを作成して
```

### Claude が行うこと

1. ドラフトを読み込み、評価能力を**ドメイン**に分類
2. 各ドメインに**具体的なプロンプト問題**を生成
3. 5/3/1/0 の**4段階採点基準**を問題ごとに定義
4. スキーマ検証を実行して `benchmarks/customer_support_benchmark.json` を保存

### 完了報告の例

```
✅ benchmarks/customer_support_benchmark.json を生成しました

  ドメイン数 : 3
  問題数     : 9
  難易度分布 : ★★ × 3  ★★★ × 4  ★★★★ × 2

ドメイン一覧:
  - intent_recognition  「問い合わせ意図の認識」  3問
  - next_step_guidance  「次ステップの提示」      3問
  - tone_and_manner     「トーンと礼節」          3問
```

### 内容の確認

Web UI の **「Benchmark」タブ** を開き、左サイドバーから `customer_support_benchmark.json` を選択すると、生成されたプロンプトと採点基準をブラウザ上で確認できます。

気になる問題があればここで確認し、必要なら直接 JSON を編集してください。

## ④ モデル A を実行する

### Run タブで設定

1. **「Run」タブ** を開く
2. 左の **Model** で `qwen2.5-7b-instruct`（モデル A）を選択
3. 左の **Benchmark** で `customer_support_benchmark.json` を選択
4. パラメータはデフォルトのまま（Temperature: 0.7、Max Tokens: 4096）で問題なし

> **プリセットを活用**: よく使う設定は「Preset Name」に名前を入れて「Save」で保存できます。
> 次回以降はプリセットを選択するだけで設定が復元されます。

### 実行

1. 問題リストで **「All」** をクリックして全問選択
2. **「▶ Run」ボタン** を押す
3. ライブ出力エリアに各問の進行状況がリアルタイム表示される

```
[ウォームアップ中...] モデルをロード中...
[1/9] intent_recognition_01  標準的な配送問い合わせ  ★★  → 完了 (3.2秒)
[2/9] intent_recognition_02  返品意図の認識          ★★  → 完了 (2.8秒)
...
[完了] benchmark_qwen2.5-7b_20260317_103000.json  [Report]
```

4. 完了バナーに **出力ファイル名** が表示される

> 実行中にモデルを切り替えたい場合は **「■ Stop」** で中断できます。
> 途中まで完了した問題は保存されており、中断した問題から再開できます。

## ⑤ 採点する

採点は **Web UI での手動採点** と **Claude Code の `/score` スキル** の2通りがあります。

### 方法 A: Web UI で手動採点

1. **「Results」タブ** を開く
2. 左パネルから実行したファイルをクリック
3. 右パネルに各問のレスポンスと採点基準が表示される
4. 各問の **0 / 1 / 3 / 5 ボタン** でスコアを付ける
5. ボタン下の **テキストエリア** に採点コメントを記入（任意）
6. フォーカスを外すと自動保存

> **採点コメントの書き方**
> - スコア5の場合: 「正確な敬語で状況を整理し、注文番号を確認した」
> - スコア3以下の場合: 「担当部署への転送を提案したが、次の手順を具体的に伝えられなかった」

全問採点が完了すると、ファイルは自動的に `results/scored/` へ移動します。

### 方法 B: `/score` スキルで AI 採点

Claude Code に以下を入力するだけです。

```
採点して
```

`results/unscored/` にある未採点ファイルを全て自動採点し、スコアと採点コメントを付与して `results/scored/` へ移動します。

```
📋 benchmark_qwen2.5-7b_20260317_103000.json  (qwen2.5-7b-instruct)
  ├─ intent_recognition_01  標準的な配送問い合わせ  → 5/5  (注文番号の確認まで含め完全な対応)
  ├─ intent_recognition_02  返品意図の認識          → 5/5  (返品ポリシーへの誘導が正確)
  ├─ intent_recognition_03  ポイント残高への質問     → 3/5  (意図は正しいが具体的な手順が不足)
  ...
  └─ Avg: 4.11/5  [→ scored/ へ移動]
```

## ⑥ モデル B でも実行・採点する

LM Studio でモデルを切り替えます。

1. LM Studio の **「My Models」** でモデル A をアンロード
2. モデル B（`lfm2.5-1.2b`）をロード
3. Web UI の **「Run」タブ** で同じベンチマーク `customer_support_benchmark.json` を選択
4. **Model** でモデル B を選択
5. 同様に実行・採点

> **複数モデルの同時評価**
> LM Studio は1つのモデルしか同時ロードできないため、モデルごとに順番に実行します。
> 同じベンチマークに対して何モデルでも結果を蓄積できます。

---

## ⑦ /compare でモデル比較レポートを生成

2つのモデルの採点が完了したら、Claude Code に以下を入力します。

```
/compare customer_support_benchmark.json
```

または自然言語で:

```
customer_support_benchmark のモデルを比較して
```

### Claude が行うこと

1. `results/scored/` から `customer_support_benchmark.json` を使った結果を全件収集
2. 各モデルのスコア・応答時間・採点コメントを集計・分析
3. 以下を含む Markdown 比較レポートを生成

| セクション | 内容 |
|-----------|------|
| 評価概要 | 全体平均・平均応答時間・満点率・0点率を一覧 |
| ドメイン別スコア比較 | 「問い合わせ意図の認識」などドメインごとにモデルを比較 |
| モデル個別評価 | 強み・弱み・速度・出力制御・失敗パターンを詳述 |
| ユースケース別推奨 | 精度優先 / 速度優先 / バランスの3シナリオで最適モデルを提示 |
| 総合推奨モデル | 根拠つきで最優秀モデルを選定 |

4. `results/reports/compare/customer_support_benchmark_compare_YYYYMMDD.md` に保存

## ⑧ Reports タブで閲覧する

1. Web UI の **「Reports」タブ** を開く
2. 左サイドバーに `customer support benchmark — 2026-03-17` が表示される
3. クリックするとレポートが整形表示される

```
# モデル比較評価レポート — カスタマーサポート AI 応答品質ベンチマーク
生成日: 2026-03-17  ベンチマーク: customer_support_benchmark.json

## 評価概要
| モデル                | 全体平均   | 平均応答時間 | 満点率 | 0点率 |
|----------------------|-----------|------------|--------|-------|
| qwen2.5-7b-instruct  | 4.33 / 5  | 5.2s       | 78%    | 0%    |
| lfm2.5-1.2b          | 2.89 / 5  | 0.6s       | 44%    | 11%   |
...
```

見出し・テーブル・blockquote がすべて整形表示され、そのままチームへの共有資料として使えます。

## Tips

**ベンチマークを育てる**
初回のドラフトで完璧なベンチマークを作ろうとしなくて大丈夫です。実行してみてプロンプトが曖昧と感じたら JSON を直接編集し、問題を修正した上で再実行できます。

**採点コメントを活用する**
`/score` のコメント・手動入力コメントは `/compare` レポートの「代表的な失敗パターン」として自動で引用されます。詳細なコメントを入れるほど比較レポートの質が上がります。

**問題を選択して再実行する**
Results タブの詳細パネルで各問の右上にある **🔄 ボタン** を押すと、その問題だけを再実行できます。プロンプトを修正した後の確認などに便利です。

**比較対象を増やす**
`/compare` はベンチマークに紐づく採点済み結果が何件あっても比較できます（2件以上）。モデルを追加するたびに再実行すれば、レポートが更新されます。
